## Semantic HTML

## What it is

Semantic HTML means choosing tags based on meaning instead of visual appearance. For example, use `header`, `main`, `section`, `article`, and `footer` when those meanings match the content.

## Why it matters

- Helps humans read the structure faster.
- Helps screen readers understand page regions.
- Helps agents retrieve cleaner structural signals from markup.

## Common errors

- Replacing every structural element with `div`.
- Missing labels for form inputs.
- Missing `alt` text for informative images.

## Verification steps

1. Check whether the page outline is understandable from headings and sections.
2. Check whether interactive controls have accessible labels.
3. Check whether decorative and informative images are distinguished correctly.
